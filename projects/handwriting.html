Coded writing

I built a handwriting engine in JavaScript that converts text into manipulatable paths, enabling morphing and animation, and have used it in multiple physical and web-based works.

I've integrated this with procedural text generation, exploring the gap between meaning and nonsense, in a space conceptually and technically adjacent to LLMs.

Defining the letters

Each letter is defined by a small number of points, which means the file size for the handwriting code is relatively small and defining new letters is quick. The points are curved into smooth paths using Chaikin's curve algorithm and then turned into a shape, so they can have varied width.

Ligature logic

Some pairs of letters can be easily joined while others do not automatically work nicely together. Here are two examples of problematic letter pairs, creating a path through the centre of the "a" and a bump between the "t" and "i".
Problematic letter pair "na"
Problematic letter pair "ti"
I assigned each letter one of four join types for the beginning and end of the path - no join, baseline, just above baseline and x-height. Then, where necessary, I gave each letter rules for how it should join to the other join types.
